# Car Market Arbitrage Analytics — описание проекта для резюме

> Pet-проект уровня production: система предиктивной аналитики, которая находит
> недооценённые объявления на рынке подержанных авто и подсказывает выгодные
> сделки. End-to-end: сбор данных → валидация → ML-оценка справедливой цены →
> скоринг сделок → дашборд + Telegram Mini App, работающий в проде 24/7.

---

## 1. Суть проекта одним абзацем

Сервис собирает реальные объявления о продаже автомобилей с auto.ru, по каждой
машине рассчитывает **справедливую рыночную цену** с помощью ML-модели (CatBoost)
и сравнивает её с ценой объявления. Если машина продаётся заметно дешевле своей
справедливой цены — это кандидат на **арбитражную сделку** (купить дёшево —
перепродать по рынку). Каждая найденная сделка ведёт на настоящую страницу
объявления. Система работает автономно: фоновый шедулер каждые 15 минут
докачивает свежие объявления по всей России и автоматически переобучает модель.

**Зачем (бизнес-ценность):** автоматизирует ручной труд перекупщика/аналитика —
вместо просмотра сотен объявлений вручную человек сразу получает отсортированный
список недооценённых машин с оценкой выгоды, надёжности предсказания и
объяснением, почему цена считается заниженной.

---

## 2. Что делает (функционал)

- **ML-оценка справедливой цены** каждого авто по 19 признакам.
- **Arbitrage Score** — взвешенный рейтинг сделки: ценовая недооценка +
  ликвидность марки + динамика снижения цены − штраф за пробег.
- **Грейды сделок:** 🔥 Горячая / 👍 Хорошая / ⚠️ Подозрительная, с уровнем
  надёжности предсказания (зависит от объёма данных по модели).
- **Детектор аномалий** (IsolationForest) — отсекает «ловушки» (скрученный
  пробег, подозрительно дешёвые объявления), отделяя реальный арбитраж от мусора.
- **SHAP-объяснения** — для каждой машины видно, какие признаки и как повлияли на
  оценку (нативные CatBoost ShapValues, без внешней зависимости).
- **История цен** — отслеживание изменения цены: продавец, снижающий цену,
  мотивирован продать быстро → это повышает score.
- **Калькулятор оценки авто** — ввёл параметры машины → получил справедливую
  цену, диапазон ±MAPE, вердикт по цене продавца и топ похожих объявлений.
- **AI-проверка надёжности** — по выбранной машине LLM ищет типичные болячки
  конкретного двигателя/поколения на пробеге и выдаёт структурированный вердикт
  (балл 0–10, слабые места, чек-лист осмотра).
- **Дашборд** (Streamlit + Plotly): обзор рынка, поиск сделок, аналитика
  качества модели, калькулятор.
- **Telegram Mini App + бот** — мобильный интерфейс (поиск/подбор/проверка),
  которым владелец пользуется ежедневно; работает в проде по постоянному
  публичному HTTPS-URL.

## 3. Метрики (на реальных данных auto.ru)

| Метрика | Значение |
|---|---|
| Объём данных | ~10 900 объявлений (train 8057 / val 2015) |
| География | вся Россия, 780+ городов |
| **MAPE модели** | **~14.6%** (стартовала с 40% → улучшена в 2.7 раза) |
| RMSE | ~1.2 млн ₽ (рынок цен от 35 тыс. до 50 млн ₽) |
| Топ-признаки | объём двигателя, мощность, тип кузова, марка/модель |
| Покрытие тестами | 119 unit-тестов в 9 файлах |

**Как улучшалась MAPE (история оптимизации):**
40.3% (база, таргет в рублях) → 27.2% (логарифмирование таргета) → 24.4%
(подбор гиперпараметров через 5-fold CV) → **14.6%** (рост датасета ×6 + по-сегментная
калибровка по пробегу).

Ключевые приёмы, давшие прирост качества:
- **Лог-таргет** `log(price)` — рынок от 35 тыс. до 50 млн ₽; в рублях RMSE
  доминировали дорогие авто и забивали ошибку на массовом сегменте.
- **Сегментация брендов** (luxury/premium/mass/budget/china) — редкие марки
  (13 объявлений Bentley) объединяются с ценовыми «соседями».
- **По-сегментная калибровка по пробегу** — депрессия цены с пробегом сильно
  отличается у бюджетного и премиум-сегмента; калибровочные коэффициенты
  считаются на валидации и применяются на инференсе.
- **Аккуратная фильтрация выбросов** — цена обрезается по квантилям в разрезе
  brand+model, но высокий пробег НЕ режется (это и есть сигнал амортизации).

## 4. Под капотом (как устроено технически)

### Поток данных (data pipeline)
```
auto.ru → scraper (aiohttp+BS4) → Pydantic-валидация → SQLite (+ price history)
        → DataPreprocessor (dedup, выбросы, feature engineering)
        → CatBoost train → predict + Arbitrage Score + SHAP + IsolationForest
        → Streamlit-дашборд + Telegram Mini App
```

### Формула Arbitrage Score
```
Score = w1·(P_pred − P_act)/P_pred + w2·L + w3·price_drop − w4·mileage_penalty
        (w1=0.8, w2=0.2, w3=0.1, w4=0.15 — настраиваются в config.py)
```
- `P_pred` — справедливая цена по ML; `P_act` — фактическая цена объявления.
- `L` — коэффициент ликвидности марки: data-driven перцентиль глубины рынка
  brand+model, масштабированный в диапазон 0.85–1.2 (с ручными оверрайдами).
- `price_drop` — доля снижения цены с первого наблюдения.
- `mileage_penalty` — штраф за большой пробег, чтобы убитые машины не лезли в топ.

### Признаки модели
- **Категориальные (10)** — отдаются в CatBoost напрямую, без ручного кодирования:
  марка, модель, кузов, КПП, регион, привод, тип топлива, сегмент бренда,
  модификация, поколение.
- **Числовые инженерные (9)** — возраст авто, пробег/год, мощность/литр
  (power density), владельцев/год и т.д., считаются в `DataPreprocessor`.

### Сбор данных (scraping)
- auto.ru рендерит карточки на сервере → хватает `aiohttp + BeautifulSoup`
  без headless-браузера.
- Селекторы по regex-префиксам классов (хэш-суффиксы билда меняются).
- Антибот: ретраи (tenacity), семафор-конкурентность, ротация User-Agent,
  паузы между регионами, проба на капчу перед прогоном; страницы с капчей
  пропускаются (не обходятся).
- Дедупликация и трекинг истории цен в БД.

### Автономность
- **APScheduler** каждые 15 минут забирает свежайшую ленту по РФ
  (сортировка freshest-first) и **авто-переобучает** модель после накопления
  порога новых объявлений — без участия человека.
- Деплой через **launchd** на macOS, постоянный публичный HTTPS для Mini App
  через **Tailscale Funnel**.

### Инженерные практики
- Конфиг как единый источник правды (`config.py`), всё переопределяется через env.
- Логирование вместо print, type hints + Google-style докстринги.
- **119 unit-тестов** (схемы, парсеры на реальном HTML-фикстуре, препроцессинг,
  скоринг, storage), линтер ruff.
- **CI (GitHub Actions):** lint → тесты → smoke-скрапинг → smoke-обучение.
- **Docker Compose:** `init (seed+train) → dashboard + scheduler` одной командой;
  образ python:3.11-slim, non-root, healthcheck.

> авто на auto.ru: async-скрапинг ~10 900 объявлений, CatBoost-модель оценки

**Маркеры достижений (bullet points):**
- Построил полный data pipeline от async-скрапинга и Pydantic-валидации до
  ML-инференса и продакшн-дашборда на ~10 900 реальных объявлений по 780+ городам.
- Снизил ошибку модели оценки цены **с 40% до 14.6% MAPE** (×2.7) за счёт
  лог-таргета, сегментации брендов, по-сегментной калибровки по пробегу и
  подбора гиперпараметров через 5-fold CV.
- Разработал композитный **Arbitrage Score** (недооценка + ликвидность +
  динамика цены − штраф за пробег) и детектор аномалий (IsolationForest),
  отделяющий реальные сделки от «ловушек».
- Сделал модель интерпретируемой через **SHAP**: для каждой машины видно вклад
  каждого признака в оценку.
- Внедрил **автономный шедулер** (сбор каждые 15 мин + авто-переобучение) и
  выкатил сервис в прод 24/7 (Docker, launchd, Tailscale Funnel, Telegram Mini App).
- Обеспечил инженерное качество: **119 unit-тестов**, линтер, CI-пайплайн.
