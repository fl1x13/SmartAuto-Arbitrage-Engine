# Как запустить проект

## ⚡ Шпаргалка (скопируй и запусти)

```bash
cd ~/Pet_project_fable

# ЗАПУСТИТЬ ВСЁ (фоном, терминал можно закрыть):
SCRAPER_BASE_URL="https://auto.ru" SCRAPER_MAX_PAGES=10 \
  nohup python3 -m scraper.scheduler > /tmp/scheduler.log 2>&1 &
nohup python3 -m streamlit run app/app.py --server.headless true > /tmp/dashboard.log 2>&1 &
# → открыть http://localhost:8501

# ОСТАНОВИТЬ ВСЁ:
pkill -f scraper.scheduler; pkill -f "streamlit run app/app.py"

# ПЕРЕЗАПУСТИТЬ = остановить, потом запустить (две команды выше)

# ПРОВЕРИТЬ ЧТО РАБОТАЕТ:
tail -5 /tmp/scheduler.log        # планировщик качает объявления?
curl -s -o /dev/null -w "%{http_code}\n" http://localhost:8501   # 200 = дашборд жив
```

## 🚀 Полный запуск приложения (дашборд + автообновление базы + модель)

Открой **два терминала** в папке проекта:

```bash
# Терминал 1 — автообновление: каждые 15 мин свежие машины с auto.ru,
# модель переобучается сама после 50 новых объявлений. Не закрывать!
SCRAPER_BASE_URL="https://auto.ru" SCRAPER_MAX_PAGES=10 python3 -m scraper.scheduler
```

```bash
# Терминал 2 — дашборд → http://localhost:8501
python3 -m streamlit run app/app.py
```

Всё. Дальше ничего делать не нужно: scheduler сам качает новые объявления и
переобучает модель, дашборд сам перечитывает базу раз в 5 минут.
Остановить: Ctrl-C в обоих терминалах.

То же самое одной командой через Docker (нужен запущенный Docker Desktop;
перед этим останови локальные процессы — порт 8501 общий):

```bash
open -a Docker            # если Docker Desktop не запущен
docker compose up         # остановить: Ctrl-C или `docker compose down`
```

## Полный цикл: скрапинг → обучение → дашборд

```bash
# Скрапинг реального auto.ru (вся Россия, свежие объявления первыми)
SCRAPER_BASE_URL="https://auto.ru" python3 -m scraper.runner --pages 5

# Переобучить модель на новых данных
python3 -m model.train

# Запустить дашборд
python3 -m streamlit run app/app.py
```

## Запуск через Docker (production-режим)

Docker нужен для деплоя на сервер. Для локальной проверки — не обязателен.

```bash

open -a Docker

# Поднять весь стек одной командой:
# 1. init-контейнер: скрапинг + обучение модели
# 2. dashboard на http://localhost:8501
# 3. scheduler: свежие объявления каждые 15 минут + авто-переобучение
docker compose up
```

## 📱 Telegram-бот (мониторинг с телефона)

1. Создайте бота у [@BotFather](https://t.me/BotFather): `/newbot` → скопируйте токен.
2. Запустите:

```bash
TELEGRAM_BOT_TOKEN="123456:ABC-ваш-токен" python3 -m bot.main
```

Что умеет:
- **🔥 Топ сделок** — лучшие недооценённые машины с фото и ссылками
- **🎯 Подбор машины** — бюджет и марка кнопками → топ вариантов
- **Оценка по ссылке** — пришлите боту ссылку на объявление auto.ru,
  он ответит карточкой со справедливой ценой и выгодой
- **🔔 Подписка** — пуш-уведомление, когда сканер находит новую горячую
  сделку (проверка каждые `BOT_WATCH_INTERVAL_MINUTES`, по умолчанию 15 мин)

В Docker: `TELEGRAM_BOT_TOKEN=... docker compose --profile bot up`

### 🚀 Mini App (приложение внутри Telegram)

Красивый интерфейс с лентой сделок, подбором и оценкой — открывается кнопкой
прямо в чате с ботом.

```bash
# 1. Запустите сервер мини-аппа
python3 -m bot.webapp                      # http://localhost:8050

# 2. Telegram требует HTTPS — поднимите туннель (любой из):
cloudflared tunnel --url http://localhost:8050   # бесплатно, без аккаунта
# или: ngrok http 8050

# 3. Перезапустите бота с публичным URL туннеля:
TELEGRAM_BOT_TOKEN="..." WEBAPP_URL="https://<ваш-туннель>" python3 -m bot.main
```

После этого у бота появится кнопка меню «Авторадар» и кнопка
«🚗 Открыть Авторадар» в /start. Опционально: `WEBAPP_REQUIRE_AUTH=1`
включает проверку подписи Telegram initData на API.

## ✅ Откуда берутся данные

**Проект работает на РЕАЛЬНЫХ данных auto.ru.** В `data/car_market.db` — 8000+ настоящих
объявлений из 500+ городов России (16 крупных городов собраны отдельными лентами),
собранных парсером `scraper/autoru.py`; ссылки
в дашборде ведут на реальные страницы auto.ru. Модель обучена на этих данных и учитывает
город как признак (цены в Москве и Владивостоке разные).

### Как собрать/обновить реальные данные:

```bash
# Один прогон: 5 свежайших страниц по всей России (~185 объявлений)
SCRAPER_BASE_URL="https://auto.ru" python3 -m scraper.runner --pages 5
python3 -m model.train
```

Выдача сортируется **по свежести** (`sort=cr_date-desc`) и берётся по **всей России**
(`SCRAPER_REGIONS=rossiya`, город каждого объявления сохраняется в колонку `region` и
доступен в фильтре дашборда). Нужны конкретные города — перечисли слаги:
`SCRAPER_REGIONS=moskva,sankt-peterburg,ekaterinburg`. Повторные прогоны записывают
изменения цен в `price_history` (сигнал «продавец мотивирован»).

### Автообновление данных (рекомендуемый режим):

```bash
SCRAPER_BASE_URL="https://auto.ru" python3 -m scraper.scheduler
```

Каждые **15 минут** (настраивается через `SCRAPE_INTERVAL_MINUTES`) скрапятся первые
5 страниц свежей выдачи, а после накопления **50 новых объявлений** модель автоматически
переобучается — справедливые цены всегда отражают текущий рынок.

**Почему 15 минут:** явно недооценённая машина в крупном городе уходит за 30–60 минут
(звонки начинаются почти сразу). Интервал 15 мин ловит такие объявления в первые минуты
жизни. Чаще нет смысла: ~25 запросов/час безопасно для антибота auto.ru, а 5-минутный
интервал утроил бы нагрузку почти без выигрыша. Дашборд сам перечитывает БД раз в 5 минут
(TTL кэша) — новые машины появляются без перезапуска.

### Ограничения реального источника:
- **Антибот auto.ru**: после ~30–40 страниц подряд отдаёт капчу — эти страницы просто
  пропускаются (логируется `showcaptcha`). Не качай много за раз; scheduler короткими
  частыми прогонами накапливает данные естественным образом.
- **`owners_count`** не показывается в карточке листинга (только на странице объявления) —
  парсер ставит 1 для всех.
- **Дата публикации** не рендерится в карточке — записывается время скрапинга.

### Demo-режим (без интернета):
Если `SCRAPER_BASE_URL` пуст, скрапер генерирует мок-листинги локально (`scraper/demo_source.py`)
и гоняет их через тот же BS4-парсинг. Синтетическая БД сохранена в `data/car_market_synthetic.db.bak`.

## Переменные окружения

Скопируй `.env.example` → `.env` и заполни:

```bash
cp .env.example .env
```

| Переменная | По умолчанию | Смысл |
|-----------|-------------|-------|
| `DATABASE_URL` | `sqlite:///data/car_market.db` | БД (можно PostgreSQL) |
| `SCRAPER_BASE_URL` | пусто (demo-режим) | `https://auto.ru` для реальных данных |
| `SCRAPER_REGIONS` | `rossiya` | Регионы: вся Россия или `moskva,sankt-peterburg,...` |
| `SCRAPER_DELAY_SECONDS` | `1.5` | Задержка между запросами |
| `SCRAPER_MAX_PAGES` | `5` | Страниц свежей выдачи за прогон |
| `SCRAPER_CONCURRENCY` | `5` | Параллельных запросов |
| `SCRAPE_INTERVAL_MINUTES` | `15` | Интервал автообновления шедулера |
| `RETRAIN_MIN_NEW_ADS` | `50` | Порог авто-переобучения модели |
